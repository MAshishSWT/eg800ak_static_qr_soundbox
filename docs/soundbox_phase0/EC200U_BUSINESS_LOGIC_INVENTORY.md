## 2. Business behavior extracted

### 2.1 Payment receive flow

From `mqtt_demo.c`:

- MQTT callback receives a payload.
- Three payload classes exist:
  - `conf#<command>#<sessionId>#<value>`
  - `play#<language>#<audioName>`
  - `<amount>#<language>#<provider>`
- Payment payload updates:
  - current amount queue/string list
  - selected language
  - selected provider
  - daily sum
  - daily transaction count
  - day transaction circular buffer
- Audio state machine later consumes the transaction and plays amount prompt.

Migration decision:

- Keep the product behavior.
- Replace delimiter parser with a bounded parser and Phase 1 JSON schema.
- Store amounts as paise integer, not `double`.
- Add transaction ID idempotency before updating totals.

### 2.2 Commands

From `mqtt_demo.c` command response logic:

- `GET_APN`
- `GET_CSQ`
- `SET_DMSURL`
- `CMD_REBOOT`
- `GET_DMSURL`
- `GET_FWVER`
- `SET_FWVER`
- `SET_AUDVER`

Migration decision:

- Keep command set as compatibility commands.
- Add command schema version, nonce/timestamp, signature, replay protection, and allowlist.
- Move command execution to `sb_command_service`.
- HTTP/HTTPS command response must be generated by `sb_http_service`, not raw string sockets.

### 2.3 Health packet

From `publishHealthStatus()`:

- Serial / IMEI.
- Model number.
- Firmware version.
- Network strength.
- Battery percent.
- Last amount.
- HTTP POST to DMS health endpoint.

Migration decision:

- Keep health content but output JSON through `ql_http_client` POST.
- Add schema version, uptime, MQTT state, data call state, daily totals, reset reason, audio pack version.
- Never build HTTP requests with unsafe unbounded string replacement.

### 2.4 Audio prompt flow

From `audio_prcs.c`, `transaction_play.c`, `summary_play.c`, `dailylog_play.c`:

- Power on prompt.
- Device setup/ready prompts.
- SIM/network/internet/MQTT error prompts.
- Battery and signal announcements.
- Transaction amount announcements.
- Daily summary announcements.
- Last transaction log announcements.
- Provider/language path composition.

Migration decision:

- Preserve language grammar and asset naming where compatible.
- Refactor into token sequence engine.
- Use fixed-size arrays and bounded string formatting.
- Centralize audio queue and serialize playback in `sb_audio_task`.

### 2.5 Key action behavior

From `keypad_prcs.c` and `AppControlPrcs()`:

- Volume up/down in normal mode.
- Mode key toggles transaction log browsing.
- In mode, up/down browse transaction log.
- Long press actions:
  - battery percentage announcement
  - network/signal announcement
  - daily summary announcement

Migration decision:

- Keep behavior.
- Implement key debounce and long-press classification in `sb_key_action_service` using GPIO EINT or periodic scan.
- Use events rather than a global `key_code`.

### 2.6 Daily reset

From `CheckDayChangePrcs()`:

- Read RTC date.
- Compare with stored last date.
- Reset daily transaction count and daily sum when date changes.

Migration decision:

- Keep behavior.
- Use `ql_rtc_get_time()` after NTP/NITZ sync.
- Store reset date and totals in A/B config or ledger summary.
- Avoid reset if time invalid; retry after time sync.

### 2.7 Firmware and audio OTA

From `fota_ftp.c` and `device_config_prcs.c`:

- Firmware update request comes from command.
- Audio pack update request comes from command.
- Status is reported back by command response.

Migration decision:

- Replace FTP-based and EC200U-specific FOTA with EG800AK `ql_fota.h` flow and HTTP/HTTPS manifest.
- Add hash/signature validation architecture.
- Use staging for audio pack OTA.

### 2.8 SMS recovery

From `sms_demo.c`:

- SMS callback receives text.
- Parser extracts phone number and message.
- Commands can be handled over SMS.

Migration decision:

- Keep as optional recovery channel.
- Disabled by default in production.
- Require enable flag, sender allowlist, command signature/password, and audit event.

## 3. EC200U implementation risks to fix during migration

| Risk | Location | EG800AK production fix |
|---|---|---|
| Unsafe `strcat()`/`strcpy()` into fixed buffers | Many files | Use bounded safe string APIs only. |
| Payload parsing modifies callback payload directly using `strtok()` | `mqtt_demo.c` | Copy into bounded buffer; use reentrant parser; validate length. |
| Money stored/processed as `double` | `mqtt_demo.c`, audio grammar files | Use integer paise. |
| No transaction ID idempotency | `mqtt_demo.c` | Add ledger with unique transaction ID and duplicate suppression. |
| Global mutable flags across tasks | `common_prj_def.*`, `mqtt_demo.c`, `audio_prcs.c` | Replace with typed event queues and state owners. |
| Hard-coded certificates/URLs/APN | `mqtt_demo.c`, `common_prj_def.h` | Use secure credential store and signed provisioning. |
| Raw HTTP request string building | `http_prcs.c`, `publishHealthStatus()`, `CommandReplay()` | Use `ql_http_client` with headers and upload callback. |
| FOTA flow tied to EC200U FTP APIs | `fota_ftp.c` | Use EG800AK `ql_fota.h` and secure manifest. |
| NVM uses EC200U `ql_cust_nvm_*` | `common_prj_def.c` | Use A/B config files plus `ql_securedata` for secrets. |
| External NOR uses EC200U `ql_api_spi6_ext_nor_flash` | `common_prj_def.c` | Use EG800AK `ql_spi_nor.h` and file system APIs validated on KAE8. |

## 4. Reusable logic by destination module

| Destination module | EC200U logic to reuse | What must be rewritten |
|---|---|---|
| `sb_payment_service` | Payment flow and language/provider selection | Parser, validation, idempotency, integer money. |
| `sb_audio_prompt_service` | Audio states and phrase sequence concepts | QuecOpen audio API calls, safe paths, playback callback handling. |
| `sb_audio_phrasebook` | Language-specific sequence rules | Data-driven token tables, paise conversion, bounded output. |
| `sb_ledger_service` | Daily circular buffer concept | Persistent ledger record format and duplicate detection. |
| `sb_key_action_service` | Key-mode behavior | GPIO/EINT handling, debounce, event generation. |
| `sb_command_service` | Command set and response concepts | Signature validation, policy, schema, HTTP response routing. |
| `sb_config_manager` | App DB concept and defaults | Magic/version/CRC/A-B slots/migration. |
| `sb_ota_policy_service` | Firmware/audio version command flow | Manifest, download, verify, apply, rollback. |
| `sb_sms_recovery_service` | SMS command channel concept | Disabled-by-default secure recovery workflow. |
| `sb_amount_format` | Indian amount formatting concept | Integer paise handling and token engine. |

## 5. Mandatory source migration rule

Do not copy EC200U source files directly into the EG800AK package. Extract behavior and rewrite using EG800AK APIs from `ql_application_eg800ak.zip`.
