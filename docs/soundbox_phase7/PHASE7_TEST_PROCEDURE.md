# Phase 7 Test Procedure

## Boot validation

Expected boot lines:

```text
[SB][I][app] starting 1.0.0-phase7-business-logic
[SB][I][business] task started
[SB][I][ledger] ready count=<n> daily_count=<n>
```

## Payment test

Publish this payload to `mqtt_sub_topic`:

```json
{"tx_id":"T1001","amount_paise":12500,"provider":"paytm"}
```

Expected:

```text
[SB][I][payment] payment accepted amount_paise=12500
[SB][I][supervisor] payment accepted amount_paise=12500
```

The device should enqueue an amount audio announcement and publish a payment ack.

## Duplicate test

Publish the same payload again.

Expected:

```text
[SB][I][payment] duplicate tx ignored
[SB][I][supervisor] payment duplicate text=duplicate
```

No second audio announcement should be generated for the duplicate transaction.

## Key test

- Short volume up: increases volume by 5%.
- Short volume down: decreases volume by 5%.
- Short mode: announces last transaction amount.
- Long volume up: announces signal status prompt.
- Long volume down: announces battery status prompt.
- Long mode: announces daily total amount or ready prompt if empty.

## Command test

`ping` is accepted without signature:

```json
{"cmd":"ping","request_id":"R1"}
```

Signed commands require a valid signature generated by the configured command validation policy. Test rejection with an invalid signature:

```json
{"cmd":"set_volume","request_id":"R2","volume":80,"signature":"00000000"}
```

Expected:

```text
[SB][W][supervisor] command rejected status=-24 text=set_volume
```

## Daily reset test

Set the module RTC date forward by one day or wait for date rollover. The service should reset daily count/total when RTC date changes.
